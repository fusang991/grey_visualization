# 从"差分与积分"理解格雷码公式

> G = B ⊕ (B >> 1)

对应的是一种特定路径，也就是**反射格雷码路径**。并不是任何哈密顿环都服从这个公式，数学还没懒惰到把所有路径压进一个公式里。

---

## 一、普通二进制和格雷码分别表示什么

假设有 3 位。

普通二进制 `B = b₂b₁b₀` 表示你沿路径走到了第几步：

```
000, 001, 010, 011, 100, ...
```

格雷码 `G = g₂g₁g₀` 表示这一步实际位于立方体的哪个顶点。

| 路径编号 (B) | 顶点坐标 (G) |
|:---:|:---:|
| 000 | 000 |
| 001 | 001 |
| 010 | 011 |
| 011 | 010 |
| 100 | 110 |
| 101 | 111 |
| 110 | 101 |
| 111 | 100 |

所以转换公式是在回答：

> **已知我走了第 B 步，现在位于超立方体的哪个顶点？**

---

## 二、为什么要"反射"

3 位格雷码由 2 位格雷码构造。

2 位格雷码：`00, 01, 11, 10`

先在最高位为 0 的子立方体中正向走：

```
000, 001, 011, 010
```

再在最高位为 1 的子立方体中反向走：

```
110, 111, 101, 100
```

注意第二半是第一半的**反向**。

因此，当你进入最高位为 1 的区域时，后面的行走方向会被翻转。

这就是公式中异或的根源：

> 上一级是否反向，会影响下一级坐标应该保持还是翻转。

---

## 三、从递归路径理解公式

公式可以逐位写成：

```
gₙ = bₙ           （最高位照抄）
gᵢ = bᵢ₊₁ ⊕ bᵢ   （其余位）
```

**最高位**：`gₙ = bₙ`

最高位直接决定当前在超立方体的哪一半：

- `bₙ = 0`：前半段
- `bₙ = 1`：后半段

所以最高位直接复制。

**下一位**：`gᵢ = bᵢ₊₁ ⊕ bᵢ`

- `bᵢ`：原本准备选择哪个子区域
- `bᵢ₊₁`：当前这一层的行走方向是否被反转
- `gᵢ`：反转修正后的实际坐标

如果 `bᵢ₊₁ = 0`，当前方向没有反转：`gᵢ = 0 ⊕ bᵢ = bᵢ`

如果 `bᵢ₊₁ = 1`，当前方向已经反转：`gᵢ = 1 ⊕ bᵢ = b̄ᵢ`

所以异或实际上是在表达：

> **实际方向 = 原选择 ⊕ 是否反向**

---

## 四、以 B = 101 为例

```
B      = 101
B >> 1 = 010
G      = 101 ⊕ 010 = 111
```

逐位看：

```
g₂ = b₂           = 1
g₁ = b₂ ⊕ b₁      = 1 ⊕ 0 = 1    （最高层在后半段，第二层方向翻转）
g₀ = b₁ ⊕ b₀      = 0 ⊕ 1 = 1
```

所以路径编号 `101`，对应顶点 `G = 111`。

---

## 五、为什么逆转换是连续异或

正向转换记录"相邻层之间是否发生方向变化"：`gᵢ = bᵢ₊₁ ⊕ bᵢ`

格雷码可以看成一种**变化记录**：

- `gᵢ = 0`：相邻两层的状态相同
- `gᵢ = 1`：相邻两层的状态不同

已知变化记录，要恢复原来的状态，从最高位一路累积：

```
bₙ = gₙ            （最高位照抄）
bᵢ = bᵢ₊₁ ⊕ gᵢ    （逐级累积）
```

- `gᵢ = 0`：保持前一层状态
- `gᵢ = 1`：翻转前一层状态

例如 `G = 111`：

```
b₂ = g₂         = 1
b₁ = b₂ ⊕ g₁    = 1 ⊕ 1 = 0
b₀ = b₁ ⊕ g₀    = 0 ⊕ 1 = 1
```

所以 `111(G) → 101(B)`

---

## 六、用"差分与积分"记忆

这是最好记的视角。

### 二进制转格雷码：做差分

```
gᵢ = bᵢ₊₁ ⊕ bᵢ
```

异或可以理解成模 2 意义下的减法。格雷码记录相邻二进制位是否不同。

> **口诀：最高位照抄，后面相邻异或。**

### 格雷码转二进制：做累积

```
bᵢ = bᵢ₊₁ ⊕ gᵢ
```

从最高位开始，逐层恢复。

> **口诀：最高位照抄，格雷位为 0 就保持，为 1 就翻转。**

---

## 七、最终理解

| | 含义 |
|---|---|
| **普通二进制 B** | 我沿反射格雷路径走了多少步 |
| **格雷码 G** | 我当前位于超立方体的哪个顶点 |

由于反射格雷路径是递归构造的，每进入一个新的子立方体，都可能反向行走。

- 正向 `gᵢ = bᵢ₊₁ ⊕ bᵢ`：利用上一级的方向，修正当前层的实际坐标
- 反向 `bᵢ = bᵢ₊₁ ⊕ gᵢ`：根据每一层的翻转记录，逐层恢复路径编号

所以真正值得记住的不是两坨公式，而是：

> **正向：局部差分**
>
> **反向：逐级累积**

公式只是这两个动作被异或符号压缩后的版本。
